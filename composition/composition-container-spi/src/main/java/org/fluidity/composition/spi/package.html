Service provider interfaces to be used / implemented by the dependency injection container implementation.

The entry point to the container implementation is a class implementing the {@link org.fluidity.composition.spi.ContainerProvider} interface that the implementation must provide. As long as the implementation is built using Maven and the org.fluidity.maven:maven-composition-plugin Maven plugin is used to build the implementation, no action is necessary for Fluid Tools to find the ContainerProvider other than packaging it in a JAR file and making sure that JAR file is loaded by the host application.

The {@link org.fluidity.composition.spi.OpenComponentContainer} returned by the ContainerProvider should extend {@link org.fluidity.composition.spi.EmptyComponentContainer}.

The implementation is offered a few basic services in the form of a {@link org.fluidity.composition.spi.ContainerServices} object given to the ContainerProvider whenever it is invoked.
